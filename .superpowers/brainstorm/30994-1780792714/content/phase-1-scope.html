第一阶段怎么切：地图情报网络 + 内容生成管线

你已经判断 1 和 3 先做。这里先确认“地图情报”如何进入内容管线，避免一开始做成孤立地图或孤立写作工具。

模块关系草图
1. 全球地图情报网络

地点 / 公司 / 事件 / 来源

标记航天港、研发中心、制造基地、测试场、投资机构、创业公司、发射/政策/事故等情报点。

2. 数据看板

融资 / 股价 / 里程碑

第二阶段补上结构化金融和上市公司数据，用来解释公司变化与行业趋势。

3. 内容生成管线

选题 / 调研 / 文案 / 分镜

从情报点触发选题，生成事实卡、角度卡、脚本、分镜、配音稿和视频任务。

建议的 MVP 闭环：地图情报点 → 选题候选 → 事实包/来源包 → 脚本大纲 → 短内容分镜。

第一阶段的地图情报，应该先偏哪种工作方式？

A

混合：编辑标注为主，AI 自动补全

团队手动确认关键点位和情报，AI 帮忙提取来源、生成摘要、建议选题和分镜。适合先做内部可用产品。

B

人工情报地图优先

先做高质量编辑数据库和地图视图，把每个地点、公司、事件的关系沉淀清楚，内容管线晚一点自动化。

C

自动信号采集优先

先接新闻、RSS、公告、投融资数据库等信号源，自动在地图上生成候选情报点，再让编辑筛选。
