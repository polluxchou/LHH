公司/项目追踪对象需要哪些字段？

第一版不要做成复杂 CRM。目标是让每天 web search 有足够上下文，能判断哪些变化值得进入地图和内容池。

一个追踪对象的最小档案
基础身份

名称、别名、国家/地区、官网、公司/项目类型、主赛道

地图关系

总部、发射场、制造设施、测试基地、产业园/政策节点

搜索配置

关键词、排除词、语言、地区、优先来源、搜索频率

内容意图

为什么追踪、关注问题、适合短内容/长内容、重要性等级

每天更新时，系统应该优先抓哪类变化？

A

商业与融资变化

融资、上市、订单、客户、并购、破产、裁员、重大合作。适合解释公司走势。

B

技术与项目里程碑

试车、首飞、发射、入轨、回收、载荷交付、产品发布。最适合航天内容。

C

地点与设施变化

新工厂、扩产、试验基地、航天港许可、产业园入驻、土地/环保/监管文件。

D

政策与监管变化

发射许可、出口管制、政府采购、补贴、制裁、频谱/遥感/通信许可。

E

人物与团队变化

创始人动态、高管离职、招聘变化、核心科学家/工程团队流动。

建议

第一版建议全选，但在内容池里给 B 技术/项目里程碑 和 C 地点/设施变化 更高权重，因为它们最容易和地图、分镜、视频叙事结合。
