开发阶段与 3 个 Agent 分工

推荐先做 4 个开发阶段。Phase 0 由主线统一完成数据模型和脚手架，之后 3 个 agent 可以并行。

阶段安排
Phase 0

基础骨架

项目脚手架、数据库 schema、共享类型、权限占位、测试框架。

Phase 1

情报追踪闭环

追踪对象库、每日 web search、信号抽取、来源去重。

Phase 2

编辑筛选闭环

编辑简报 Inbox、内容价值评分、通过/观察/拒绝、选题池。

Phase 3

地图与验收

地点锚点、地图上下文、端到端验收、种子数据、部署。

Tracking Object
Search Run
Candidate Signal
Source

Agent 1：追踪与搜索

负责公司/项目追踪对象、搜索配置、每日 web search、结果聚类去重、B/C/D 信号分类。

Editorial Brief
Content Value Score
Screening Decision
Topic Card

Agent 2：简报与筛选

负责 AI 编辑简报、内容价值评分、筛选动作、状态流转和选题池生成。

Location Anchor
Map Context
UI Feedback
E2E Flow

Agent 3：地图与体验

负责地点锚点、地图上下文、工作台 UI、点击确认反馈、端到端可用性。

关键约束

三个 agent 不能各自发明数据结构。Phase 0 必须先冻结共享 schema、状态枚举、API 合约和测试种子数据，然后再并行。
